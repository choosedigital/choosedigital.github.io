Choose Digital REST API
for Developers

Choose Digital is a proud sponsor of the MIA Music Summit 2014!

The Choose Digital API provides all the methods needed for a full-featured, all-digital-media online store: from browsing and search, to complete product data, to transactions and order history. This can be done with relatively basic API calls to Choose Digital servers.

CodePlay, our gift code product, is an example of a general large-scale digital media store. It was built and runs entirely on the API.

/music/artist/band%20of%20horses/us/true?rows=2

Authentication

Every request to the API must contain authentication information to establish the identity of the principal making the request. In REST, this is done by first putting the headers in a canonical format, then signing the headers using your Choose Digital Secret Access Key.

Access keys for testing (add as request parameters):

cdaccesskeyid: 8627E307937657A8F29B0152378DF6C6 authorization: 6VEcIRl5rq7HGRqTqhEdcxQTJtulcgW3HrVpuQS9oGp7Abjy5SPodymIpczAjnY

The current date-time is also required in UTC format in the request headers, for example:
x-cd-date: 2014-02-03T09:40:00.0000Z
(do not use this date, always pass the current date)

More about Authentication in the docs.

Don’t assume

  1. Always parse response data without assuming the number or order of elements in the response. For example, if a new field were to be added to the object your code should gracefully handle a change in the total number of elements, or a change where the element you are interested is now 5th instead of 4th.
  2. Always use hostnames in your code and let the DNS servers resolve the IP — don’t pre-resolve the name in order to try and save time at runtime, as the IP will not always be the same.
